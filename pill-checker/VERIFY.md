## 開発環境の動作確認手順（30%ステップ）

1. **依存関係のインストール**
   ```bash
   npm install
   ```
   初回セットアップ時のみ必要です。  

2. **開発サーバー起動**
   - Next.js 16.0 を利用しているため、Webpack モードで起動します。
   ```bash
   npm run dev -- --webpack
   ```
   - もし 16.0 より前のバージョンを使う場合は通常の `npm run dev` で問題ありません。  

3. **ブラウザで表示確認**
   - `http://localhost:3000` を開き、以下を確認します。
     - ユーザー切替ボタンが2人分表示される
     - 時間帯ボタンが5種類表示され、クリックでハイライトが変わる
     - 選択した条件に応じて「用意する薬の内訳」が更新される
     - フォームからユーザー/薬/スケジュールを追加すると、上部の表示に即反映される
     - 「カメラ仮判定」セクションでカメラ起動→プレビューが表示される（許可が必要）
     - 認識数を変更して「判定を実行」を押すと、不足/過剰/未登録のメッセージが表示される  
   - iPhone 実機もしくはシミュレーターで同画面が崩れないか確認すると、PWA 化の下準備にもなります。  

4. **Lint チェック**
   ```bash
   npm run lint
   ```
   エラーが出なければ現状のコード品質は Next.js の推奨規約を満たしています。  

以上の項目が通れば、30% ステップ（骨組み）の動作確認は完了です。
