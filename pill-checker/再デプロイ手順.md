# 再デプロイ手順（GitHubにプッシュしてVercelで再デプロイ）

## 📋 このガイドについて

このガイドでは、修正したコードをGitHubにプッシュして、Vercelで自動的に再デプロイする方法を説明します。

**所要時間**: 約5-10分

---

## ✅ 前提条件

- [ ] GitHubアカウントを持っている
- [ ] GitHub Desktopがインストールされている
- [ ] Vercelでプロジェクトが作成済み
- [ ] VercelとGitHubが連携済み

---

## 🚀 手順

### ステップ1: GitHub Desktopでプロジェクトを開く

1. **GitHub Desktopを起動**
   - Windowsのスタートメニューで「GitHub Desktop」を検索して起動

2. **プロジェクトを開く**
   - メニューバーから「File」→「Add Local Repository」をクリック
   - 「Choose...」ボタンをクリック
   - 以下のフォルダを選択：
     ```
     C:\Users\t-ich\Documents\いまにゅClaud AI Cording202507\合宿202511\お薬飲み忘れツール\pill-checker
     ```
   - 「Add repository」をクリック

3. **既にリポジトリが追加されている場合**
   - GitHub Desktopの左側のリポジトリ一覧から`pill-checker`を選択
   - または、メニューバーから「File」→「Open Local Repository」をクリックして選択

---

### ステップ2: 変更をコミット（記録）する

1. **変更内容を確認**
   - GitHub Desktopの左側に、変更されたファイルのリストが表示されます
   - すべてのファイルが選択されていることを確認（チェックボックスがオン）

2. **コミットメッセージを入力**
   - 左下の「Summary」欄に、変更内容を入力
   - 例：「デプロイ設定を修正」「Root Directory設定を追加」など

3. **コミットを実行**
   - 「Commit to main」ボタンをクリック
   - 数秒待つと、コミットが完了します

---

### ステップ3: GitHubにプッシュ（アップロード）する

1. **リモートリポジトリの確認**
   - GitHub Desktopの上部に「Push origin」ボタンが表示されているか確認
   - 表示されていない場合：
     - メニューバーから「Repository」→「Repository settings...」をクリック
     - 「Remote」タブをクリック
     - 「Primary remote repository」の「Remote URL」に、GitHubリポジトリのURLを貼り付け
       - 例：`https://github.com/neco3bucchan-ship-it/pill-checker.git`
     - 「Save」をクリック

2. **プッシュを実行**
   - 「Push origin」ボタンをクリック
   - 数秒待つと、GitHubにアップロードされます
   - 「Pushed to origin」と表示されたら成功です ✅

---

### ステップ4: Vercelで自動デプロイを確認

1. **Vercelのダッシュボードを開く**
   - ブラウザで https://vercel.com/dashboard を開く
   - ログインしていることを確認

2. **プロジェクトを開く**
   - プロジェクト一覧から、デプロイしたプロジェクト名をクリック
   - 例：`pill-checker` や `pill-checker-4` など

3. **デプロイの状態を確認**
   - プロジェクトページの上部タブから「Deployments」をクリック
   - 最新のデプロイの状態を確認：
     - ⏳ **「Building」** → ビルド中（2-3分待つ）
     - ✅ **「Ready」** → デプロイ成功！
     - ❌ **「Error」** → エラーが発生（ログを確認）

4. **デプロイが成功した場合**
   - 「Ready」と表示されたら、デプロイ成功です
   - その行の右側に「Visit」ボタンまたはURLが表示されます
   - URLをクリックするか、コピーして保存してください
   - 例：`https://pill-checker-xxxxx.vercel.app`

---

### ステップ5: 404エラーが出る場合の対処法

404エラーが表示される場合は、以下を確認してください：

1. **Root Directoryの設定を確認**
   - プロジェクトページの「Settings」タブをクリック
   - 左側のサイドバーから「Build and Deployment」をクリック
   - ページを下にスクロールして「Root Directory」セクションを見つける
   - 「Edit」ボタンをクリック
   - `pill-checker` と入力して「Save」をクリック

2. **再デプロイを実行**
   - 「Deployments」タブに戻る
   - 右上の「Redeploy」ボタンをクリック
   - 「Redeploy」をクリックして再デプロイを開始
   - 2-3分待つ

3. **再度確認**
   - デプロイが「Ready」になったら、再度URLにアクセス
   - 正常に表示されることを確認

## ❓ よくある質問

### Q: 「Push origin」ボタンが表示されない

**A**: リモートリポジトリが設定されていない可能性があります。ステップ3の1を参照して、リモートリポジトリを設定してください。

### Q: プッシュ時にエラーが発生した

**A**: 以下を確認してください：
1. GitHubにログインしているか
2. リモートリポジトリのURLが正しいか
3. インターネット接続が正常か

### Q: デプロイが自動的に開始されない

**A**: 以下を確認してください：
1. VercelとGitHubが正しく連携されているか
2. GitHubにコードがプッシュされているか
3. Vercelの「Deployments」タブで最新の状態を確認

### Q: エラーが発生した

**A**: ビルドログを確認してください。よくある原因：
- Root Directoryの設定が間違っている（`pill-checker`に設定する必要がある）
- 依存関係のエラー（`package.json`の問題）
- TypeScriptのエラー
